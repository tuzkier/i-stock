# 路由前置设计决策（为什么按意图、且无条件先路由）

本文记录入口胶囊「路由前置」段与 `skill-router` 触发判据的设计意图，避免后续维护者在不了解背景时把它改回旧形态。

## 它修的是什么病

观察到的真实故障：**coding agent 在新会话里根本没走 `skill-router` 就直接执行**。一路取证后定位的根因不是"约束不够"、不是"复杂度判断不准"、也不是 router 内部的覆盖缺口，而是更上一层：

> `skill-router` 的"每轮自动触发"是一种**约定**，没有真正的触发器。它依赖 agent 在动手前**自愿**先调用一次。对"看起来就该立即执行"的任务，coding agent 的**动作先验**（拿到活就干）压过这条提示词，于是 router 从未被调用——它内部的 `check_new` / 复杂度 / intake 矩阵全都不会触发。

也就是说：治理是在**路由那一刻**丢的，不是在"写文件"那一刻。任何 router 内部的改动都救不了"router 没被调用"。

## 决策

在**入口胶囊**（`package/adapters/codex/AGENTS.md` 的「路由前置」段；`CLAUDE.md` 经 `@AGENTS.md` 继承）植入一道**前移的、提示词层的强制前置**，三个机制叠加：

1. **无条件第一动作**：删掉"需要判断时再路由"这个由 agent 自判的条件，改为每条消息的第一句话固定用自然语言说清这次怎么接（这是什么任务、走哪个 skill 还是直接响应、依据一句话），必须先于任何工具调用 / 代码 / 文件改动。把路由从"可选前置"变成"必须先说出来的第一件事"。**只约束"先判断、且显式说出来"，不规定固定模板**——早期版本用过 `[路由] 键=值` 的填表式声明，因读起来机械、像在走形式而退役；要保的是判断本身被说出来、被人当场看见，不是某个格式。
2. **专家身份框定**：把默认身份从"拿到需求就写代码的 coding agent"重定义为"资深技术负责人"——路由是专业本能，不是流程开销。正面身份框定比"你不是 coding agent"的否定句更能对冲动作先验。
3. **跳过必须留痕**：判"直接响应"必须当场写明依据，把沉默绕过变成显式、可被人当场抓到的绕过。

`skill-router` 侧同步：description 增加"执行意图"触发条件（CSO 合规，仍是纯触发条件）、红线表增加对应一行。三处落点分别覆盖"还没进 router / 匹配 router / 进了 router 之后"。

## 关键约束：按"意图"判，不按"关键词"判

判据是**"用户是不是在要求立即把某件事实现 / 落地 / 改掉"这个意图**，不是某几个字面词。

- ✅ 正确：`用户要求立即把某件事做出来 / 落地 / 改掉` → 必须先路由。
- ❌ 错误：`用户说了"立即执行 / 直接改 / 按文档落地"这几个词` → 退化成关键词匹配，换种说法就漏，且会误伤只是带了"执行"二字的提问。

「立即执行」「按 XX 文档落地」等只能作为该意图的**举例**，不能作为判据本身。

## 同样关键：执行意图 ≠ 跳过路由的授权

用户"要求马上做"是**执行意图 / 执行确认**（对应 intake「意图本体识别」里的 `execution_confirmation`）——它确认"要做这件事"，恰恰把类型钉成 `新任务 / 继续 / 缺陷`，因此**更要求先路由**，绝不免除路由。哪怕用户给了文档或需求作为输入，落地路径仍由 `skill-router` 决定，不得把"按文档做"当成"跳过 Harness 约束开干"。

## 已知边界（诚实交底）

这套全部是**提示词层**的软约束，**提高命中率、不保证 100%**。一个对抗性或已严重上下文衰减的 agent 仍可能无视注入文本。要把"router 没被调用"变成结构上不可能，唯一彻底的办法是 runtime 侧的 **hook**（如每轮 `UserPromptSubmit` 注入、或 `PreToolUse` 在改文件前 deny），那超出入口胶囊范围，按需另行决策。本决策是"在不引入 hook 的前提下，最强的软版本"。

## 不要这样改回去

- 不要把"路由"改回"需要判断时 / 觉得需要时"再触发——那个由 agent 自判的条件正是被取消掉的逃逸口。
- 不要把意图判据换成字面关键词清单。
- 不要把"用户说了立即执行"当成可以跳过路由的授权。
- 不要把那句自然语言路由声明改回固定的 `[路由] 键=值` 填表模板——软化外壳是刻意的：要保的是"判断被显式说出来、被看见"这件事，不是它穿什么格式。但也别走到另一极端把声明整个省掉——"显式说出来"本身不可省。
