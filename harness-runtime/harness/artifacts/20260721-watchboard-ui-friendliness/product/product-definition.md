## 业务定义

**业务问题：** 多市场看盘终端界面「像调试台」而非「给人用的看盘工具」——正常态被误导性警告色渲染、内部技术枚举（`trend_state`/`score_band` 等）与理由代码直接暴露在主视图、同一信息（来源状态、价格/涨跌）在 3~5 处重复、顶部大标题被周期/视图控件抢焦点、主看信号与回测流水账权重接近、侧栏主看信息与来源状态混排、交易信号卡三段回测默认铺开。用户打开界面需付出过高认知成本才能扫读主看信息，且易被误导性视觉信号干扰。

**业务目标：** 完成 7 类呈现层友好化改造，使看盘用户（ACT-01，唯一主参与者，只读观察者）打开界面即可低成本扫读主看信息（标的、价格、涨跌、当前可操作信号视觉突出），正常态不触发警告色，内部技术枚举不暴露主视图，同一信息不重复呈现——即「让界面忠实呈现既有领域状态、不误导、不泄漏内部编码」。

**成功信号（可观察 / 可量化）：**
- 正常态（来源 formal/not_loaded、恢复 restored/partial/default_fallback、评分 positive/neutral）主看盘 **0 处黄色警告横幅（`.data-notice`）**；真异常态（来源 stale/unavailable、恢复 failed/坏布局丢弃）**可见警告色**（SCN-01）。
- 主视图 **0 处裸枚举、0 处裸理由代码**；评分**有进度条式呈现**（SCN-02）。
- 来源状态、价格/涨跌各**收敛到 1 处权威主源（计数=1）**；侧栏来源仅为弱化小圆点次级信号（SCN-03）。
- 顶部标题字号/权重**高于**周期/视图控件且控件右对齐（SCN-04）；主卡与次级信息**存在明确视觉层级差**（SCN-05）。
- 侧栏每条名称+代码一行、价格+涨跌右对齐（SCN-06）；交易信号卡**默认仅关键数字、三段回测默认折叠**（SCN-07）。
- `npm run build` exit 0（EVD-01，前提证据）；7 类改造点逐条 preview 截图比对（EVD-02）；Playwright E2E + axe 无回归（EVD-03）。

**任务契约匹配度：** 完全匹配。目标、成功定义、非目标、约束、验证口径在 mission-contract 中清晰且相互一致（scope-strategy「授权边界结论」证实无授权边界缺口）。唯一未定项为 `demo_fallback` 来源色档（DEC-01 / SCOPE-10），落为待用户确认的产品决策，默认信息级、不阻断任务成立，不构成 NEEDS_DECISION。

## 当前流程摘要

看盘用户在本地运行的看盘终端应用中，从侧栏选中某自选标的（ACT-01 只读），系统（ACT-02）每次观测重算来源健康（STM-02）、MTS 技术解释（OBJ-03）、交易信号（STM-05）、回测/反T（OBJ-08）、工作台恢复态（STM-07）等既有领域状态，界面据此渲染。当前渲染存在 7 类呈现缺陷（见问题诊断）。本次改造只调整「领域状态 → 界面呈现语义」这一读侧映射，不改任何领域计算。改造后用户扫读路径不变，但呈现忠实、无误导、无泄漏。

## 业务用例模型

### BUC-01: 盯盘并判断某标的当前状态与可操作信号

**业务参与者：** ACT-01 看盘用户

**业务目标：** 打开某标的，低成本判断「数据可不可信、技术面偏强还是偏弱、现在该不该动」，不被误导性警告色或裸枚举干扰。

**触发条件：** 用户选中 / 切换到某自选标的。

**前置业务状态：** 该标的在自选列表中（active）；ACT-02 已重算其来源健康 / MTS / 交易信号。

**主业务流程：**
1. 用户切换到某标的，界面呈现该标的行情（价格/涨跌）与来源健康档。
2. 用户扫读技术信号解释（趋势/评分/提醒级别/理由），判断技术面强弱。
3. 用户查看交易信号卡（建议/关键价位/关键回测数字），判断是否可操作。

**扩展 / 异常业务路径：**
- 来源降级（stale/unavailable/demo_fallback）：用户需一眼识别「数据是否可信」，且不被正常降级误导为严重故障。
- 数据不足 / 非策略标的：交易信号不可用，用户需理解「为何没有信号」而非看到裸枚举。
- 技术面偏空（negative 评分 / 风控级）：用户需识别这是「市场业务结论」而非「系统异常」。

**后置业务结果：** 用户获得对该标的当前状态与可操作信号的正确认知，不被误导性警告色或裸枚举干扰。

**关联业务对象 / 规则：** OBJ-01/02/03/04/07/08；BR-02/03/04/05/07。

### BUC-02: 维护并快速扫读自选标的列表

**业务参与者：** ACT-01 看盘用户

**业务目标：** 在侧栏自选列表中快速定位目标标的，扫读名称/代码/价格/涨跌等主看信息。

**触发条件：** 用户查看侧栏列表。

**前置业务状态：** 自选列表含多条标的（active/archived）。

**主业务流程：**
1. 用户扫视侧栏列表条目。
2. 每条呈现名称+代码为主行、价格+涨跌为主看数字。
3. 来源状态弱化为次级信号（小圆点），不与主看信息混排。

**扩展 / 异常业务路径：** 某标的来源降级——条目来源弱化信号需能区分正常/需关注，但不喧宾夺主。

**后置业务结果：** 用户以低成本定位目标标的，主看信息突出、来源状态弱化不抢视线。

**关联业务对象 / 规则：** OBJ-01（STM-01）、OBJ-02；BR-02。

### BUC-03: 感知工作台恢复与布局状态

**业务参与者：** ACT-01 看盘用户

**业务目标：** 打开工作台时正确感知「布局是否正常恢复」，仅在真异常时被提醒。

**触发条件：** 应用启动，ACT-03 读取快照产出恢复态；用户切换布局模式。

**前置业务状态：** 存在 restoreMetadata（STM-07 四态之一）；布局模式为 STM-06 之一。

**主业务流程：**
1. 应用启动，ACT-03 读取快照，ACT-02 判定恢复态四态之一。
2. 界面据恢复态呈现：正常恢复保持中性/成功语义。
3. 用户可切换布局模式（dense/focus/mobile_tab），呈现随之调整（不改布局语义）。

**扩展 / 异常业务路径：** 恢复失败（failed）或坏布局被丢弃——以警告/需关注语义提示。

**后置业务结果：** 正常恢复（restored/partial/default_fallback）不触发警告色误导；仅 failed/坏布局丢弃时提示需关注。

**关联业务对象 / 规则：** OBJ-06（STM-06/STM-07）；BR-10、BR-11。

### SUC-01: 查看标的行情与来源健康状态

### SUC-02: 查看技术信号解释（MTS 人话化 + 评分可读）
**参与者：** ACT-01 | **目标：** 判断技术面偏强还是偏弱、为什么，不需懂内部枚举 | **覆盖业务用例：** BUC-01
**主成功流：** 查看技术提醒区 → 读取 displayLabel/technicalReminder/scoreBand/alertLevel/reasons → 人话文案+进度条评分，理由显示 label/detail → 按 BR-04 对评分极性上色（与来源故障色分离）→ 原始枚举/理由代码收进可折叠详情。
**备选流：** 偏空结论（negative/风控）→ 谨慎/风险色路径；展开详情 → 原始字段可见。 **异常流：** data_insufficient → 人话「数据不足」；source_degraded → 仅解释性提醒+来源子集。
**涉及对象/状态/规则：** OBJ-03、OBJ-07；BR-04、BR-05。 **关联系统操作：** SUC-02-OP-01/02。 **关联验收：** SCN-02-COND-01/02/03/04、SCN-01-COND-06。 **是否需要 UI 承载：** 是（UIC-01/UIC-02）。

### SUC-03: 查看交易信号卡（关键数字 + 明细折叠）
**参与者：** ACT-01 | **目标：** 一眼抓到「现在该不该动」的结论与关键回测数字 | **覆盖业务用例：** BUC-01
**主成功流：** 查看信号卡 → 读取 status；ready 时读 stanceLabel/holding/levels/回测关键数字 → 主卡突出建议+关键数字（胜率/累计收益）→ 三段回测/回合/事件/原始字段降为次级并默认折叠 → nonAdvice 免责保持可见。
**备选流：** 持仓/空仓 → 对应 stance；展开明细 → 三段回测/反T回合/价位分层。 **异常流：** not_target_symbol/data_insufficient/source_degraded → 各自人话化，无裸枚举；source_degraded 无回测块（BR-07）。
**涉及对象/状态/规则：** OBJ-04（STM-05）、OBJ-08（STM-08）；BR-03、BR-07。 **关联系统操作：** SUC-03-OP-01/02。 **关联验收：** SCN-05-COND-01/02、SCN-07-COND-01/02/03、SCN-02-COND-04。 **是否需要 UI 承载：** 是（UIC-02/UIC-05/UIC-07）。

### SUC-04: 扫读侧栏自选列表
**参与者：** ACT-01 | **目标：** 在自选列表快速定位标的，扫到名称/代码/价格/涨跌 | **覆盖业务用例：** BUC-02
**主成功流：** 扫视侧栏 → 每条读名称/代码/价格/涨跌/来源档 → 名称+代码一行、价格+涨跌右对齐呈现主看 → 来源档弱化为小圆点（次级），不作来源状态主源。
**备选流：** archived 标的 → 弱化呈现，区分 active。 **异常流：** 条目来源降级 → 小圆点表达需关注，不喧宾夺主。

### SUC-05: 顶部标的切换与周期 / 视图操作

| 方案需要判断的输入 | 来源 | 对路线选择的影响 | 不能改变 / 不能假设 | 不清楚时怎么处理 |
|--------------------|------|------------------|----------------------|------------------|
| 核心系统用例 | SUC-01~06（全为呈现型读侧用例） | 方案须覆盖 7 类改造点 + 跨呈现一致性；所有用例为只读映射，无写入路线 | 不得把呈现型用例改写为新增系统行为；不得新增系统用例 | 缺口回流产品定义；不自行发明系统责任 |
| 系统操作 | SUC-01-OP~SUC-06-OP（全为读侧映射） | 方案落成「领域状态→呈现语义」映射层；操作间依赖（来源→信号降级、ready→回测、恢复态↔来源态分离）是关键约束 | 不得在呈现层重实现 domain 门控（BR-03 已在 domain 层）；不得打破操作间依赖 | 呈现层只读 STM-02/05/07 结果并保持三处一致 |

| 质量与运行约束 | 本文「质量与运行约束」5 项 | 跨呈现一致性、状态色语义分离、免责可见、fallback 明示、可访问性无回归须落进方案 | 不得为实现便利牺牲一致性或隐藏免责 | 一致性门控只读承载；免责/fallback 明示为合规红线不可省 |
