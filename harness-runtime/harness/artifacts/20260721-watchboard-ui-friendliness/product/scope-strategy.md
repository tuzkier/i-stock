> **用途**：基于任务契约、用例模型（SUC-01~06 / UIC-01~07 / SUC-xx-OP / DEC-01）、验收场景（SCN-01~07、22 条件、NEG-01~07）、业务对象（OBJ-01~09 / STM-01~08 / BR-01~12）和风险，判断范围内、范围外、延后和需要决策的事项。本文不选择技术路线，不拆执行任务。

## 模板约定

- 范围项编号固定使用 `SCOPE-xx`，范围决策使用 `IN` / `OUT` / `LATER` / `DECISION_NEEDED`。
- 每个范围判断均引用业务用例、系统用例、界面承载要求、验收场景 / 条件、业务规则、风险或任务契约。
- 范围取舍均说明业务价值、风险、证据、任务契约追溯、验证影响和对方案路线的影响。
- 不默认压缩范围；不为实现便利切断核心业务闭环。
- 不适用项写 `不适用：原因...`，证据不足项写 `待澄清：原因...`。
