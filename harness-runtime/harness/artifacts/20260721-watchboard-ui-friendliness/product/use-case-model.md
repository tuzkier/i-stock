## 模板约定

- 用例编号固定使用 `BUC-xx`（业务用例）、`SUC-xx`（系统用例）、`SUC-xx-FLOW-xx`（系统用例流步骤）、`SUC-xx-OP-xx`（系统操作）、`UIC-xx`（界面承载要求）、`DEC-xx`（待决策问题）；流步骤扇出的结局状态节点（节拍）在流步骤上附结局状态标识（如 `SUC-01-FLOW-02.formal`）。
- 严格区分三层：**SUC** = 一个参与者目标 + 系统边界内达成它的全部 flow；**flow** = 一条路径（主 / 备选 / 异常）；**state** = 一个结局 / 条件；**节拍(step)** = 流步骤 × 结局状态。flow 与 state 多对多，严禁把「备选流」等同于「一个状态」。
- 呈现型用例说明：本任务无写状态迁移的系统操作；`SUC-xx-OP-xx` 描述的是「系统读取既有领域状态并将其映射为界面呈现语义」这一读侧行为，写入列一律为「不适用：呈现层不写领域状态」。
- 任何空项都必须写 `不适用：原因...` 或 `待澄清：原因...`，不得留空。
- 用例展开中新发现的对象 / 状态 / 迁移 / 规则或术语冲突必须写入 `## 领域模型反馈`，不得在本文件里静默重写业务对象分析。

## 业务用例模型

### BUC-01：盯盘并判断某标的当前状态与可操作信号

| 项 | 内容 |
|----|------|
| 业务参与者 | ACT-01 看盘用户 |
| 业务目标 | 打开某标的，低成本判断「数据可不可信、技术面偏强还是偏弱、现在该不该动」 |
| 触发条件 | 用户选中 / 切换到某自选标的 |
| 前置业务状态 | 该标的在自选列表中（active）；ACT-02 已重算其来源健康 / MTS / 交易信号 |
| 后置业务结果 | 用户获得对该标的当前状态与可操作信号的正确认知，不被误导性警告色或裸枚举干扰 |
| 涉及业务对象 | OBJ-01 WatchSymbol、OBJ-02 SourceHealth、OBJ-03 MtsExplanation、OBJ-04 TradeSignalState、OBJ-07 MtsReason、OBJ-08 回测/反T |
| 业务规则 | BR-02（来源三档）、BR-03（降级门控信号）、BR-04（评分负向≠异常）、BR-05（理由人话化）、BR-07（ready 才有回测） |
| 证据 | US-01/02/03/05/07；SCN-01/02/03/05/07；OBJ-01 REF-03 |

**主业务流程：**
1. 用户切换到某标的，界面呈现该标的行情（价格 / 涨跌）与来源健康档。
2. 用户扫读技术信号解释（趋势 / 评分 / 提醒级别 / 理由），判断技术面强弱。
3. 用户查看交易信号卡（建议 / 关键价位 / 关键回测数字），判断是否可操作。
4. 用户据以形成判断，必要时展开次级明细核对。

**扩展 / 异常业务路径：**
- 来源降级（stale/unavailable/demo_fallback）：用户需一眼识别「数据是否可信」，且不被正常降级误导为严重故障。
- 数据不足 / 非策略标的：交易信号不可用，用户需理解「为何没有信号」而非看到裸枚举。
- 技术面偏空（negative 评分 / 风控级）：用户需识别这是「市场业务结论」而非「系统异常」。

### SUC-01：查看标的行情与来源健康状态

| 项 | 内容 |
|----|------|
| 参与者 | ACT-01 看盘用户 |
| 参与者目标 | 打开某标的即判断「现在这份数据可不可信 / 是不是最新」，并读到价格与涨跌 |
| 来源业务用例 | BUC-01 |
| 来源系统责任 | 按 BR-02 分档呈现来源色（formal/not_loaded 中性、demo_fallback 信息级、stale/unavailable 警告）；价格 / 涨跌与来源状态各收敛唯一主源 |
| 前置条件 | 已选中某标的；ACT-02 已产出 SourceHealth（STM-02 之一）与行情观测 |
| 触发 | 用户选中 / 切换标的，或触发刷新 |
| 后置结果 | 用户正确识别来源可信度；正常态不见警告色，仅真异常见警告色；价格 / 涨跌只在唯一主源呈现 |
| 涉及对象 / 状态机 / 规则 | OBJ-02（STM-02）、EXC-01 价格/涨跌属性；BR-02、BR-03（门控波及提示） |
| 触发状态迁移 | 不适用：呈现层只读 STM-02，不驱动迁移 |
| 关联验收场景 / 条件 | SCN-01、SCN-03 |
| 是否需要界面承载 | 是，原因：来源健康与行情是纯视觉承载，色语义与去重复是核心验收点（UIC-01/UIC-03） |

**主成功流：**
1. 用户切换到某标的。
2. 系统读取该标的 SourceHealth 档位与最新行情观测。
3. 系统按 BR-02 将来源档映射为呈现色语义并在唯一主源呈现来源状态。
4. 系统在唯一主源呈现价格 / 涨跌（涨跌红绿与来源警告色语义分离）。

**备选流：**
- 备选流 A（降级但可用）：来源为 demo_fallback → 走信息级提示路径（DEC-01，默认不用高危警告色）。
- 备选流 B（未选中 / 初始）：无选中标的或数据未加载 → not_loaded 中性占位路径。

**异常流：**
- 异常流 X（真异常）：来源为 stale / unavailable → 走警告 / 错误色提示路径，并标注受影响对象（chart/mts/alerts）。

**状态枚举（结局 / 条件，含进入触发）：**

| 状态 ID | 状态（结局 / 条件） | 态类型 | 进入触发 | 涉及对象 / 状态机 | 可观察结果 |
|---------|---------------------|--------|----------|-------------------|------------|

| ST-01b | formal（正式健康数据） | 业务态 | 选中标的且加载成功 | OBJ-02 / STM-02.formal | 正常 / 中性色，无警告横幅 |
| ST-01c | demo_fallback（演示 / 兜底） | 业务态 | 数据不可得时回落兜底数据 | OBJ-02 / STM-02.demo_fallback | 信息级 / 次级提示（DEC-01，默认非高危警告色） |
| ST-01d | stale（数据过期） | 业务态 | formal 数据变陈旧 | OBJ-02 / STM-02.stale | 警告色 + 降级说明 + 受影响范围 |
