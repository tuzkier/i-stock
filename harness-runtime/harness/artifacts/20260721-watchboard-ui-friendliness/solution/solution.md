## 3. 候选路线比较

### 3.1 真候选判定

本任务在「系统组织方式」维度上存在一处实质分叉：**如何组织状态色语义承载**（引入语义色分层 vs 保留 `.data-notice` 逐组件内联条件）。对该分叉跑真候选判定：

- 分叉维度检查：Route A（语义色 token 分层 + 集中只读映射）与 Route B（保留共享告警类逐组件内联着色）在「系统组织方式 / 关键机制」维度实质分叉。
- 取舍指向检查：B 在「改动量 / 上手速度」略优，但 A 在「语义分离正确性 / 抗退化 / 去重复可保证性 / 范围外组件隔离」全面占优——B 无法稳健满足 INV-03（评分与来源故障物理区分）与 DRV-01（共享类跨范围内外），即 **B 在决定成败的维度被 A 支配**。
- 判定结论：B 属**被支配路线**，不列为正式候选；Route C（引入设计系统 / 组件库重做）违反 SCOPE-15 全量重构禁令，属越界路线。故本任务只有一条合理路线（Route A），B / C 按「被排除路线」处理。

### 3.2 选定路线：Route A — 语义色 token 分层 + 只读呈现映射层

`路线 = 语义色 token 分层 + 只读呈现映射层`
`承载方式 = ① 在 styles.css 建立四档语义色 token（正常 / 信息 / 谨慎-风险 / 警告-异常），承载类名互不复用；② 新增呈现映射（纯只读函数：领域状态 → 语义档 / 人话文案，复用既有 sourceStatusLabel/displayLabel/stanceLabel），POL-01~03 落地；③ 把范围内 .data-notice 用法迁移到语义档承载，RestoreStatus 按 STM-07 四档分色；④ 原始枚举 / 理由代码 / 技术态收进可折叠详情，折叠状态由组件本地 UI 状态承载；⑤ 来源 / 价格涨跌收敛唯一主源，侧栏来源降为小圆点弱化；⑥ 顶部标题主位 + 控件降级；⑦ 信号卡默认关键数字、明细折叠、nonAdvice 常驻`
`取舍 = 收益：四套状态语义各自归位可验证、抗后续退化（新增用法不会误染警告色）、范围外组件零波及、去重复可用计数断言证明；代价：前期需建立语义 token 体系与映射层，改动比逐组件补丁更系统；牺牲：不追求一次性重做整套设计系统（受 SCOPE-15 约束）`
`适用条件 = 改动限呈现层、门控已在 domain 层、存在既有人话源（均已由现状核对证实）`
`主要风险 = 语义 token 与既有样式冲突 / 遗漏某处 .data-notice 用法导致某态漏改；由技术分析枚举全部范围内承载点 + verify 逐档截图兜底`
`验证难度 = 逐档并置 preview 截图 + DOM 断言（类名归属 / 计数=1）+ E2E 折叠交互 + axe 无回归；主观项（层级差）双证据`

### 3.3 无实质备选说明（被排除路线）

| 被排除路线 | 排除原因（违反的上游约定 / 项目约束 / 风险） | 对后续实现的约束 |
|------------|-----------------------------------------------|------------------|
| Route B：保留 `.data-notice` 逐组件内联条件着色 | ① 无法稳健实现 INV-03「评分与来源故障色物理区分」（同一类名仍可能被复用）；② 共享类跨范围内外组件（DRV-01），逐组件内联仍需碰共享类，易波及范围外 AlertRulePanel/WorkbenchShell；③ 属「局部小改」，把语义分离不确定性包装成「实现时注意」，留下退化地雷 | 技术分析 / 执行**不得**以「在共享 `.data-notice` 上加条件判断」作为状态色归位实现方式；必须迁移到独立语义承载 |
| Route C：引入设计系统 / 组件库重做界面状态呈现 | 违反 SCOPE-15（全量界面重构 / 无关技术债禁令）、爆炸半径失控、超授权、与「仅 7 类锚定改造」冲突 | **不得**新增组件库依赖或重写非 7 类锚定的界面结构 |
| 全局重着色共享 `.data-notice` 类（不新增承载） | 直接波及范围外组件（AlertRulePanel 空态、WorkbenchShell / LayoutController 摘要与错误），SCOPE-15/16 越界，改变范围外用户可观察行为 | **不得**通过修改共享类的颜色定义来「一次性修好」；只迁移范围内用法到新语义承载 |

| SUC-04-OP-01（侧栏主看突出 + 来源弱化小圆点） | DEC-S05、DEC-S02 | 新建侧栏条目结构承载；来源降为小圆点弱化（非第二主源） | 保留：触发（用户扫读）不变；目标操作「主看突出 + 来源弱化」；不写；BR-02 弱化上色 + STM-01 archived 区分承载；可观察结果「主看突出、来源不重复主源、archived 弱化区分」达成 |
| SUC-05-OP-01（标题主位 + 控件降级右对齐） | DEC-S09 | 新建 / 重排顶部视觉层级承载；不改 STM-06 切换语义 | 保留：触发（观察 / 切换）不变；目标操作「标题主位 + 控件降级」；不写；SCN-04 + STM-06 语义不变承载；可观察结果「标题主位、控件右对齐、切换后标题仍主位、非法值既有归一化」达成 |

## 6. 关键决策

> 编号 `DEC-Sxx`（solution 决策，区别于产品 DEC-01）。只记录影响系统边界、关键机制、禁用做法、风险处理或技术分析方向的决策。

### DEC-S01：呈现层只读映射层，不重实现领域门控（防腐边界）
- **决策点**：跨呈现一致性、来源降级门控由谁承载。
- **上游依据**：BC-01↔BC-02 只读防腐、POL-03、INV-07、RISK-04（门控在 `trade-signals.ts:619-627`）。
- **候选与选择**：唯一合理路线——呈现层只读取既有门控结果并翻译。拒绝「呈现层重算 / 复制 BR-03 降级判断」。
- **禁用做法**：不得在 `src/App.tsx`/`features/*` 重新实现来源→信号降级门控；不得为 MTS 新建状态机；不得改 `src/domain/*`。
- **风险处理**：阻断类禁用（见 RISK-03/SR-01 之外的 DRV-02 压力），责任阶段 = 技术分析 + 执行 + 审查。
- **下游责任**：技术分析定义「读哪个 domain 输出字段、如何保证三处读同一结果」的接口方向。

### DEC-S02：四档语义色 token 分层，承载类名互不复用
- **决策点**：状态色如何归位、如何保证四套语义物理区分。
- **上游依据**：INV-01~03、SCN-01-COND-01~06、POL-01、SCOPE-01；现状 DRV-01（`.data-notice` 语义重载）。
- **候选与选择**：Route A（语义色 token 分层）选定；Route B（共享类内联着色）、全局重着色共享类被排除（见 §3.3）。
- **禁用做法**：不得沿用「非 formal 一律 `.data-notice`」「所有恢复态复用同一黄条」「negative 与来源故障共用告警色」；评分承载不得复用来源警告类名。
- **风险处理**：核心机制决策，SR-01 隔离共享类，验证兜底。
- **下游责任**：技术分析枚举四档 token 命名 / CSS 结构、范围内全部承载点清单、与既有 styles.css 冲突排查。

### DEC-S03：人话化复用既有可读源，不新建映射表
- **决策点**：枚举 / 理由代码→人话由复用还是新建映射承载。
- **上游依据**：RISK-03、DRV-04、POL-02、SCOPE-02；现状 `sourceStatusLabel`/`displayLabel`/`stanceLabel`/`nonAdvice` 已存在。
- **候选与选择**：复用既有可读源（默认）选定；拒绝新建枚举→文案映射表（双份翻译源、语义漂移）。
- **禁用做法**：不得把原始枚举 / 理由代码当展示文案直呈；不得新建重复映射表；未注册理由码回落 UNKNOWN_CODE 不直呈。
- **风险处理**：接受既有源为权威，缺口（未注册码）按 UNKNOWN_CODE 回落。
- **下游责任**：技术分析确认既有可读源覆盖全部主视图枚举点，标注确无既有源的缺口（若有）。

### DEC-S04：折叠 / 展开交互状态由组件本地 UI 状态承载（interaction 跳过补偿）
- **决策点**：interaction skippable，折叠交互状态归属。
- **上游依据**：RISK-02、DRV-05、UIC-02/07、SCN-02-COND-03 / SCN-07-COND-02。
- **候选比较（真候选，取舍互有高低）**：
  - `路线 = 组件本地 UI 状态（useState）`：承载方式 = 折叠态存组件内 state；取舍 = 实现简单、无持久化副作用、不碰 workspace 快照（符合 CODE-002 边界），代价 = 刷新后回默认折叠态；适用条件 = 折叠为纯 UI 表现态、非领域状态；主要风险 = 无；验证难度 = 低（E2E 展开 / 收起断言）。
  - `路线 = 折叠态持久化进 workspace 快照 / URL`：承载方式 = 折叠偏好写 localStorage / query；取舍 = 记住用户偏好，代价 = 触碰 OBJ-06 快照写入（越界 SCOPE-12 存储门控）、引入不在范围的持久化模型（违反 CODE-002）、放大爆炸半径；适用条件 = 需求要求记忆折叠偏好（本任务无此要求）；主要风险 = 越界改存储；验证难度 = 高。
- **选定路线**：组件本地 UI 状态（useState）。折叠是纯 UI 表现态（use-case-model ST-02e/ST-03f 已定性为纯 UI 态），无需持久化，避免触碰范围外存储。
- **禁用做法**：折叠状态不得写 localStorage / workspace 快照；不得因 interaction 跳过而丢失折叠义务（折叠入口必须可操作，不靠 URL 到达）。
- **下游责任**：技术分析定义各折叠承载组件的本地状态形状与展开 / 收起入口。

### DEC-S05：唯一主源承载点选点与侧栏来源弱化
- **决策点**：来源 / 价格涨跌「唯一主源在哪一处」、侧栏来源如何降级。
- **上游依据**：UIC-03、DMF-02、SCN-03-COND-01/02、DRV-06；现状来源状态在标题 / 图表角标 / 指标条 / 侧栏 3~5 处重复。
- **候选与选择**：本方案确定「主区保留 1 处权威来源承载 + 1 处权威价格 / 涨跌承载，侧栏来源降为小圆点弱化次级信号」的承载准则；具体主源落到哪个组件（如主卡头部 vs 顶部区）由技术分析在此准则内选点，但**必须计数=1**。
- **禁用做法**：不得在侧栏 / 标题黄条 / 图表角标 / 指标条多处重复来源或价格为同级主源；侧栏来源不得作第二主源。
- **风险处理**：承载选点决策，PRD 只固化计数=1，不派生验收色值。
- **下游责任**：技术分析指定唯一主源承载组件并给出去重后被移除 / 降级的重复呈现点清单。

### DEC-S06：跨呈现一致性只读约束，恢复态与来源态独立承载
- **决策点**：来源 stale 时三处呈现如何保持一致；恢复态与来源态是否共用承载。
- **上游依据**：INV-07、NEG-01/03、操作间依赖三条、SCOPE-09。
- **候选与选择**：呈现层读同一 domain 门控结果保持三处一致；恢复态（STM-07）与来源态（STM-02）用**独立承载元素**、不共用同一黄条。
- **禁用做法**：不得出现「来源已 stale 而信号卡仍 ready」矛盾呈现；不得让恢复态与来源态混用同一 `.data-notice` 承载（NEG-03）。
- **下游责任**：技术分析定义信号卡 source_degraded 呈现如何绑定来源门控结果、恢复与来源承载的 DOM 独立性断言点。

### DEC-S07：demo_fallback 默认信息级档，色档可切换不改承载结构
- **决策点**：demo_fallback 来源色档最终归类（产品 DEC-01 / SCOPE-10）。
- **上游依据**：产品 DEC-01、AUD-02、SCN-01-COND-02、DRV-09、RISK-01。
- **候选与选择**：默认「信息级 / 次级提示（非高危警告色）」软口径推进；承载结构设计为色档可切换（若用户改判「需关注」则并入警告档），不改承载元素结构。
- **禁用做法**：不得把 DEC-01 当硬阻断；不得三档（formal / demo_fallback / stale）共用同一视觉。
- **风险处理**：接受默认档推进，verify 前经用户确认定档（RISK-01）。
- **下游责任**：技术分析把 demo_fallback 承载设计为「档位可配置」（信息级 ↔ 警告档切换成本最小）。

### DEC-S08：nonAdvice 免责常驻承载，不进可折叠区
- **决策点**：信息层级重排 / 折叠后免责如何保持可见。
- **上游依据**：INV-06、AUD-01、PIT-003 / CODE-003、SCN-05-COND-02 / NEG-05、DRV-08。
- **候选与选择**：`nonAdvice` 作为常驻承载，默认态与折叠态均可见，不放入可折叠明细区。
- **禁用做法**：层级降级 / 明细折叠不得隐藏免责；免责不得进折叠区。
- **风险处理**：合规红线，阻断类约束。
- **下游责任**：技术分析给出免责在折叠态可见的 DOM 断言点。

### DEC-S09：顶部视觉层级重排，不改布局切换语义
- **决策点**：标题主位 / 控件降级如何承载、是否触碰 STM-06。
- **上游依据**：UIC-04、SCN-04-COND-01/02、STM-06、NEG-06、SCOPE-05。
- **候选与选择**：以视觉权重（字号 / 权重）+ 右对齐承载标题主位 / 控件降级；切换语义（STM-06 白名单）不改，非法值由既有归一化处理。
- **禁用做法**：不得改 STM-06 布局模式集合 / 切换逻辑；不得让控件与标题抢焦点。
- **下游责任**：技术分析定义顶部区结构调整点与「切换后标题仍主位」的 E2E 断言。
