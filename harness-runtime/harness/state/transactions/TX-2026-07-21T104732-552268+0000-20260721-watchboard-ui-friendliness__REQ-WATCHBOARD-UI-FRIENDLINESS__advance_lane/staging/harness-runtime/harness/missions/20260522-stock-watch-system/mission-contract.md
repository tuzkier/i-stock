## Objective

交付一个以本地网页为入口的多市场股票看盘系统，支持港股、A股、美股、韩股的自选管理、图表与技术指标展示、MTS 多周期趋势信号和买卖/风险提醒，并保留后续扩展到正式行情 API、基本面分析、回测和桌面封装的结构。

## 成功定义

- **期望效果:** 用户能够在本地浏览器里统一管理四个市场的自选股，快速查看价格趋势、技术形态与 MTS 风险/提醒，并且不依赖云账号即可持续使用。
- **主要交付物:** 本地网页看盘界面；多市场自选管理与代码归一；技术指标、MTS 信号与提醒层。
- **验证证据:** 单元与集成测试、冻结行情 fixture / 可回放样本片段、页面截图、浏览器重开后的本地持久化恢复验证。
- **非目标:** 自动交易；确定性投资建议或收益承诺；云同步或账号体系；完整基本面分析模块；在 intake 阶段锁定正式行情供应商、回测数据源和成本模型。

## 验收标准

**AC-01:** 系统能按市场管理自选股并保持代码归一。
- **Given** 用户输入来自港股、A 股、美股和韩股的原始股票代码。
- **When** 用户把这些标的加入同一个本地自选清单并重新打开页面。
- **Then** 系统保留市场分组，且代码按各市场规则保持可识别、可重载。

**AC-02:** 打开个股时能看到默认图表与可切换指标面板。
- **Given** 当前选中任意一只可显示行情的数据标的。
- **When** 用户进入该标的的看盘页面。
- **Then** 主图、成交量和默认指标组合自动渲染，并可切换副图指标。

**AC-03:** 系统能按研究设计输出 MTS 多周期趋势信号。
- **Given** 当前标的具备足够的历史行情数据。
- **When** 系统完成一次信号计算或数据刷新。
- **Then** 页面同时展示趋势状态、MTS 分数带、信号类型和提醒等级，而不是只显示单一买卖箭头。

**AC-04:** 当价格或指标触发风险条件时，系统能输出买卖/风控提醒。
- **Given** 当前标的在刷新后触发趋势破坏、突破确认或风控阈值。
- **When** 系统重新计算提醒状态。
- **Then** 页面显示相应提醒等级与触发理由，且能区分观察、确认、强信号和风控。

**AC-05:** 系统在本地浏览器环境中能恢复自选和提醒设置。
- **Given** 用户已保存自选股和提醒配置。
- **When** 用户关闭浏览器并在同一浏览器配置中重新打开应用。
- **Then** 系统从本地持久化恢复配置，无需重新登录或创建账号。
