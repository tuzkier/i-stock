## 摘要

为阿里巴巴港股 HK.09988 构建多技术指标组合的买卖点信号算法：历史与当前的买入/卖出信号直接标注在看盘终端 K 线图上，算法附可复现的历史回测证据（胜率、相对收益），并全程保持「非投资建议」边界。完成后，用户打开 HK.09988 图表即可获得系统化、可解释、可检验的技术信号参考。

## 验收条件

**验收条件：HK.09988 图表上可见历史买入/卖出信号标注**
- **下游追溯锚点** SCN-01
- **前置条件** 看盘终端已加载 HK.09988 某一支持 range 的历史 K 线，且该窗口内算法产生过信号
- **触发动作** 用户查看该图表
- **预期结果** 历史买入/卖出信号点直接标注在 K 线图上，买卖方向可区分，标注位置与对应 K 线时间对齐

**验收条件：当前信号状态可见且随数据可复算**
- **下游追溯锚点** SCN-02
- **前置条件** HK.09988 最新 K 线数据已加载
- **触发动作** 算法对含最新 K 线的数据完成计算
- **预期结果** 界面展示当前信号状态（如买入/卖出/观望），数据更新后重新计算得到一致口径的状态

**验收条件：信号由多技术指标组合规则产生，规则可追溯**
- **下游追溯锚点** SCN-03
- **前置条件** 算法实现与说明文档已产出
- **触发动作** 审查算法实现与说明文档（verification_method：人工审查 + EVD-04 测试）
- **预期结果** 信号由不少于两个技术指标的组合规则产生，指标选型、组合逻辑与参数在代码或文档中可追溯，与回测所用规则一致

**验收条件：算法附可复现的历史回测证据**
- **下游追溯锚点** SCN-04
- **前置条件** FutuOpenD 可获取的 HK.09988 最大历史 K 线窗口（事实上限由 discovery 确认）
- **触发动作** 对该窗口运行回测
- **预期结果** 输出口径明确的统计结果（至少含信号胜率与相对买入持有收益），且同数据同参数重复运行结果一致

**验收条件：信号展示保持非投资建议边界**
- **下游追溯锚点** SCN-05
- **前置条件** 信号标注与当前信号状态已在 UI 展示
- **触发动作** 用户查看信号标注、状态或其说明文案
- **预期结果** 存在明确的「技术分析提醒，非投资建议」类边界文案，且不出现荐股式 / 承诺收益式措辞

## 升级规则

- **阻断性技术风险**：FutuOpenD 数据链路失效 / HK.09988 历史深度不足以支撑有意义回测（GAP-03 取证后若窗口过短）→ 停下请求用户决策
- **范围扩展**：出现向多标的泛化、实时推送、MTS 体系改造的需求信号 → 停下发起 Decision Gate，不自行纳入
- **外部依赖失效**：FutuOpenD 或 futu SDK 环境异常导致数据不可得 → 停下报告，不用 mock 数据冒充真实回测证据
