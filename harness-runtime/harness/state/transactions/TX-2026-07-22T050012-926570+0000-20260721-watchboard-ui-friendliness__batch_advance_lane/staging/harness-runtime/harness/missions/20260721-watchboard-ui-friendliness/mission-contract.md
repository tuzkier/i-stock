## 摘要

> 一句话概括：做什么、为什么做、完成后的世界有什么不同。

把多市场看盘终端界面从「像调试台」改成「像给人用的看盘工具」：修正状态色滥用、把内部枚举人话化、消除重复信息、重排顶部、建立信息层级、优化侧栏与交易信号卡密度。完成后用户打开界面即可低成本扫读主看信息，不再被误导性警告色和原始技术字段干扰。

## 目标

> 核心目标：描述完成后系统/产品的**状态变化**，不写实现方式。

看盘终端界面完成友好化改造：状态色语义归位、内部枚举人话化/折叠、重复信息收敛到唯一主源、顶部信息重排、信息层级建立、侧栏扫读优化、交易信号卡密度优化，用户打开界面即可清晰扫读主看信息、不再被误导性警告色和原始枚举干扰。

## 成功定义

> 本节定义最终以什么效果、什么交付物、什么证据判断任务达标。

- **期望效果**：用户在看盘界面能以更低认知成本扫读关键信息——主看信息（标的、价格、涨跌、当前可操作信号）视觉突出；正常态不再触发警告色；内部技术枚举不再直接暴露在主视图；同一信息不再多处重复。
- **主要交付物**：DEL-01~DEL-07 七项界面呈现层改造（状态色 / 枚举人话化+评分可读 / 去重复 / 顶部重排 / 信息层级 / 侧栏 / 信号卡密度）。
- **交付格式**：`src/styles.css` + `src/App.tsx` + `src/features/*` 组件的 JSX/样式变更。
- **验证证据**：EVD-01 `npm run build` exit 0；EVD-02 7 类改造点 preview 截图逐条比对；EVD-03 Playwright E2E + axe 无回归。
- **非目标**：不改交易信号算法/回测计算逻辑本身、不改数据源/后端/server、不新增业务功能、不触碰 alibaba mission 在途策略代码语义。

## 验收条件

**验收条件：SCN-01 正常态不出现警告色，真实异常态才出现**
- **下游追溯锚点** SCN-01
- **前置条件** 某标的来源为 formal/已恢复、布局未回退（正常态）
- **触发动作** 用户打开该标的界面
- **预期结果** 界面无黄色警告横幅；来源变为 stale/unavailable 或布局回退时才出现警告色

**验收条件：SCN-02 主视图不直接暴露原始枚举/理由代码，评分可读**
- **下游追溯锚点** SCN-02
- **前置条件** 某标的有 trend_state/mts_score/score_band/signal_type/alert_level 及理由代码
- **触发动作** 用户查看主视图
- **预期结果** 主视图呈现人话描述与进度条式评分；原始代码仅在展开的详情/调试区可见

**验收条件：SCN-03 来源状态与价格/涨跌各收敛到唯一主源**
- **下游追溯锚点** SCN-03
- **前置条件** 改造前来源状态与价格/涨跌在 3~5 处重复
- **触发动作** 用户扫读改造后界面
- **预期结果** 来源状态、价格/涨跌各仅保留 1 处权威呈现点

**验收条件：SCN-04 顶部标题主位、周期/视图切换降级右对齐**
- **下游追溯锚点** SCN-04
- **前置条件** 用户打开某标的界面
- **触发动作** 观察顶部区域
- **预期结果** 标题为视觉主位（字号/权重最高）；周期/视图切换降级右对齐，不抢视线

**验收条件：SCN-05 主卡突出、次级信息降灰**
- **下游追溯锚点** SCN-05
- **前置条件** 某标的同时有当前可操作信号与回测/理由/原始字段
- **触发动作** 用户查看界面
- **预期结果** 主卡视觉突出；回测/理由/原始字段为次级灰字，存在明确层级差

**验收条件：SCN-06 侧栏条目主看信息突出、来源状态弱化**
- **下游追溯锚点** SCN-06
- **前置条件** 侧栏含多条自选标的
- **触发动作** 用户扫读侧栏
- **预期结果** 每条名称+代码一行、价格+涨跌右对齐为主看信息、来源状态弱化为小圆点

**验收条件：SCN-07 交易信号卡默认关键数字、细节可展开**
- **下游追溯锚点** SCN-07
- **前置条件** 某标的有卖出/买入/反T波段与三段回测
- **触发动作** 用户查看交易信号卡
- **预期结果** 默认呈现关键数字（胜率/累计收益）；三段回测明细默认折叠，展开后才呈现
