# 中途纠偏 — 航线修正

## 概述

在执行过程中发现偏差时，安全地修正方向而非盲目继续。

## 何时使用

- 实现过程中发现设计矛盾
- 范围溢出：实际工作量远超预期
- 上游文档（PRD/tech-design）有错误
- 用户改变了需求
- 发现未预见的技术障碍

## 何时不使用

- 只是缺陷修复（→ systematic-debugging）
- 正常的 TDD 迭代（→ 继续执行）

## 快速参考

| 步骤 | 产出 |
|------|------|
| 识别偏差类型 | 分类：设计矛盾/范围/需求变更/技术障碍 |
| 影响评估 | 哪些任务项受影响？需要回退多远？ |
| 决策（用户参与） | 修改设计 / 缩减范围 / 重新规划 |
| 更新文档 | 修改受影响的stage documents |
| 恢复执行 | 从修正点继续 |

按 `workflow.md` 执行详细步骤。
