# Retrospective — 复盘

## 概述

分析任务执行过程，提取经验教训，更新流程改进项。

## 何时使用

- 任务交付后自动触发
- 用户说"做复盘"、"回顾一下"
- 同样的问题反复出现
- 执行耗时明显超出预期

## 快速参考

| 步骤 | 产出 |
|------|------|
| 收集数据（commit 历史、执行日志） | 事实基础 |
| 分析好/坏/改 | 经验教训 |
| 提取 Action Items | 可执行的改进 |
| 更新 project-context.md | 持久化经验 |

按 `workflow.md` 执行详细步骤。
