# 设计规范（design spec）

> 视觉地基 + 四条横切维度（状态/反馈 · 内容/术语 · 领域对象表达 · a11y/响应式），基础组件与业务组件都建在它上面。
> 蒸馏来源：观察产品主导样式值 → 规范化成一小份 canonical 集；token **真值只存 `prototype/tokens.css` 一份**，本文件只登记 token 名（不手抄值，避免第二真相源漂移）。每条带 source / status；没抽到标「待补」，不凭通用最佳实践硬填。

## 交互约定

- 反馈分级：系统处理中 / 输入问题 / 权限限制 / 业务规则拒绝 / 下游失败 / 成功，各有区分的表现。
- 键盘与焦点：可聚焦顺序、快捷键、焦点恢复的项目统一约定。
- 危险 / 不可逆动作：确认、解释、撤销 / 恢复路径。
- 文案语言：用户可见文案默认中文；品牌名 / 代码标识 / 上游指定外文登记例外。

## 状态与反馈 canonical（横切）

> 每个组件 / page-state 的状态走统一表现，不各画各的。这是治"原型只有 happy path"的横切基线。

| 状态类 | 统一表现 |
|--------|----------|
| loading | <如：骨架屏，非 spinner> |
| empty | <如：插画 + 主 CTA 引导> |
| error | <如：就近内联 + 重试，不丢输入> |
| permission | <如：禁用 + 解释，不静默隐藏> |
| success | <如：toast / 内联确认> |

## a11y / 响应式基线

- 可访问性：键盘可达、焦点可见、对比度、必要 ARIA 的项目最低标准。
- 响应式：断点与 desktop / tablet / mobile 的自适应策略；常用页面骨架对应 `interaction/references/layout-patterns/`。
